비글본 블랙에서의 Dynamic Linking 오류

June 24, 2016
beagleboard dynamic linking linux

임베디드 시스템에선 코드 작성 및 컴파일을 빠르게 하기 위해 Host PC와 Target board를 NFS로 연결하여 Host PC에서 컴파일 작업을 하고, Target board에서는 실행시키는 구조로 개발환경을 빌드한다.

NFS 개발환경을 설정한 다음 arm gcc를 이용하여 컴파일을 했음에도 불구하고 No such file or directory 오류가 발생하였다.
계정 권한, 파일 권한, Host PC의 mount 권한까지 모두 확인하였지만 잘못된 부분이 없었기에 무척 난감했는데 Dynamic Linking이 원인이였다.

test가 실행파일이라 가정하고 file 명령어를 이용하여 확인해보자.

test: ELF 32-bit LSB executable, ARM, EABI5 version 1 (SYSV), dynamically linked, interpreter /lib/ld-linux.so.3, for GNU/Linux 2.6.16, not stripped

/lib/ld-linux.so.3는 프로그램이 메모리에 적재되는 시점에서 실행에 필요한 라이브러리를 링킹해주는 파일이다.
Host PC에서 프로그램 링킹은 ld-linux.so.3가 담당하지만 비글본 블랙에서의 링킹은 /lib/ld-linux-armhf.so.3이 담당한다.
비글본 블랙에서 ld-linux.so.3를 찾을 수 없으니 실행이 되지 않았던 것인데 그렇다고 ld-linux-armhf.so.3를 ld-linux.so.3로 바꾸면 안된다.

ld-linux-armhf.so.3를 링크하는 ld-linux.so.3 심볼릭 링크를 생성시키면 올바르게 동작한다.

sudo ln -s /lib/ld-linux-armhf.so.3 /lib/ld-linux.so.3

비글본 블랙에서 위의 명령어로 ld-linux.so.3 심볼릭 링크를 생성하고 ./test로 다시 실행해보자.
이젠 제대로 실행한다.
