Learning MSP430F5529 - # Interrupt

Sun, Jul 10, 2016 msp430f5529, gpio, timer, interrupt

이 글은 인터럽트 예제를 분석한 글이다, 어느 누군가에게 도움이 되길…

아래 코드는 ti.com에서 제공하는 인터럽트 예제코드1이다.

...

#define RED_LED P1OUT   // On-board RED led.

int main(void)
{
    P1DIR |= 0x01;      // Set direction OUT the led.
    TA0CCTL0 = CCIE;
    TA0CCR0 = 50000;
    TA0CTL = TASSEL_2 + MC_1 + TACLR;

    __bis_SR_register(LPM0_bits + GIE);
}

...

#pragma vector=TIMER0_A0_VECTOR
__interrupt void TIMER0_A0_ISR(void)
{
    RED_LED ^= 0x01;
}

GIE - General interrupt enable의 약자로 MCU의 인터럽트 허용여부를 결정한다.

LPM0_bits - CPUOFF 값에 매핑되어 있다, 해당 비트가 1로 설정되면 CPU는 off 상태로 변경된다.

__bis_SR_register() - CPU의 SR2를 제어하는 함수이다, 해당 레지스터는 CPU core에 존재하며 CPU 연산 결과에 대한 상태를 저장한다.


컴파일하고 MSP430F5529의 메모리를 디버깅해보면 __bis_SR_register 함수를 실행하고 GIE bit가 enable되는 순간부터 TA0R3 레지스터가 TA0CCR04 레지스터의 값과 비교하며 50000까지 증가한다.
상태 레지스터를 디버깅해보면 CPUOFF bit가 1로 세팅되어 있으므로 ISR5만 계속 처리하고 있음을 알게된다.

코드를 아래와 같이 변경하고 다시 실행하면 while 루틴과 번갈아가며 처리함을 알 수 있다.

...

#define RED_LED P1OUT       // on-board RED led.
#define GREEN_LED P4OUT     // on-board GREEN led.

int main(void)
{
    P1DIR |= (1 << 0);      // set direction OUT the RED led.
    P4DIR |= (1 << 7);      // set direction OUT the GREEN led.
    TA0CCTL0 = CCIE;
    TA0CCR0 = 50000;
    TA0CTL = TASSEL_2 + MC_1 + TACLR;

    __bis_SR_register(GIE);

    while (1)
    {
        int i;

        GREEN_LED ^= (1 << 7);
        for (i = 0; i < 30000; i++);
    }
}

...

#pragma vector=TIMER0_A0_VECTOR
__interrupt void TIMER0_A0_ISR(void)
{
    RED_LED ^= 0x01;
}

재미있는건 TA0R 레지스터 값이 증가할 때 전혀 CPU core 레지스터를 사용하지 않는다..!
또한 SR 전체인지, GIE bit인지는 모르겠지만 GIE가 0으로 설정되고 다른 ISR 호출을 허용하지 않는다.
