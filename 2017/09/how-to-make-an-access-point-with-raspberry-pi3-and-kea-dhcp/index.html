How to make an Access Point with Raspberry Pi3 and Kea-Dhcp

22 Sep 2017

In this post, we are taking a look at how to make an access point with raspberry pi3 and Kea-dhcp.
But I am not going to talk about Kea-dhcp in further detail.

Note that $ is the meaning of shell/prompt/user input.

Tested hardware

  • Raspberry Pi 3
  • Panda Wireless Adapter a.k.a PAU05 based on Ralink 5372 chipset and rt2800 device driver.

Why Kea?

By Eddy Winstead1,

ISC DHCP is now 21 years old

Stable, effective, but…
- Slow by modern standards
- Flat file lease storage
- Restarts likely required for config mods
- Failover only in peer arrangements
- Oh, those man pages!

One of the reasons I am into Kea-Dhcp is that, it provides hook functions for customizing.
It is awesome, isn’t it?

Install softwares

We just need to install two softwares, hostapd to turn rPI3 to Access Point and Kea-dhcp for giving an IP to clients.
Run the line to install them.

$ sudo apt update && sudo apt install -y hostapd kea-dhcp4-server

Latest version of Kea-dhcp is 1.1.0 in latest Raspberry Pi3.

hostapd configuration

Create a new file with the command sudo nano(or vi) /etc/hostapd/hostapd.conf
Paste the following codes,

interface=wlan0         # interface name for running hostapd.
#driver=rtl871xdrv      # Comment this for ralink chipset.
ssid=AP-TEST            
hw_mode=g
channel=6
macaddr_acl=0
auth_algs=1
ignore_broadcast_ssid=0
wpa=2
wpa_passphrase=raspberry    # at least 8 alphabets.
wpa_key_mgmt=WPA-PSK
wpa_pairwise=CCMP
wpa_group_rekey=86400
ieee80211n=1
wme_enabled=1

Note that you can change any options as you want.
Save it.

Move on to bind the config file to hostapd daemon.
Open /etc/default/hostapd file with your favorite editor, find the line #DAEMON_CONF="" and edit it as following,

...
# file and hostapd will be started during system boot. An example configuration
# file can be found at /usr/share/doc/hostapd/examples/hostapd.conf.gz
#
DAEMON_CONF="/etc/hostapd/hostapd.conf"

# Additional daemon options to be appended to hostapd command:-
# 	-d   show more debug messages (-dd for even more)
# 	-K   include key data in debug messages
...

Save it.

Likewise, edit /etc/init.d/hostapd.
Find the line DAEMON_CONF= and edit it as follwing,

...
DAEMON_DEFS=/etc/default/hostapd
DAEMON_CONF=/etc/hostapd/hostapd.conf
NAME=hostapd
...

Save it.

Now you need IP forwarding and open firewall to talk to your crush ;-).

Open the file /etc/sysctl.conf and find the lines as following,

...
# Uncomment the next line to enable packet forwarding for IPv4
#net.ipv4.ip_forward=1

# Uncomment the next line to enable packet forwarding for IPv6
#  Enabling this option disables Stateless Address Autoconfiguration
#  based on Router Advertisements for this host
#net.ipv6.conf.all.forwarding=1
...

Uncomment the both lines net.ipv4.ip_forward=1 and net.ipv6.conf.all.forwarding=1.
Run the lines as following,

$ sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
$ sudo iptables -A FORWARD -i eth0 -o wlan0 -m state --state RELATED,ESTABLISHED -j ACCEPT
$ sudo iptables -A FORWARD -i wlan0 -o eth0 -j ACCEPT

Note that if you use another interface, like wlan1, you should change the name of interface as you want.

Kea-dhcp configuration

Since you have installed kea-dhcp4-server, the configuration file of kea-dhcp is from /etc/kea/kea-dhcp4.conf.
Open it, find the lines and edit it as following,

...
# Add names of interfaces to listen on.
  "interfaces-config": {
    "interfaces": [ "wlan1" ]  # As you want, wlan1 is for me.
  },

...

# Use Memfile lease database backend to store leases in a CSV file.
  "lease-database": {
    "type": "memfile",      # Type of lease, it can be a host of database as MySQL, MariaDB.
    "persist": true,
    "name": "/var/lib/kea/dhcp4.leases", 
    "lfc-interval": 1800
  }

...

# also a list of structures.
  "subnet4": [
  # Edit subnet information as you want.
  {     "subnet": "192.168.100.0/24",
        "pools": [ { "pool": "192.168.100.10 - 192.168.100.200" } ] }
  ]
},

...

That is not enough yet, we have to set the address of the router up for clients that communicate between them.
Now time to append router setting as following to the config file in Dhcp4": { ... }.

...

# Global (inherited by all subnets) lease lifetime is mandatory parameter.
  "valid-lifetime": 4000,

# Domain name
  "option-data": [
      {
          "name": "domain-name-servers",
          "data": "8.8.8.8, 8.8.4.4"
      },
      {
          # Add router information.
          "name": "routers",
          "data": "192.168.100.1"
      }
  ],

...

Now restart the dhcp server with the command as below,

$ sudo service kea-dhcp4-server restart
$ sudo service kea-dhcp4-server status

Then you should see as following,

Interface configuration

As following lines are for my setting of the interface, hope it helps you.

# interfaces(5) file used by ifup(8) and ifdown(8)

# Please note that this file is written to be used with dhcpcd
# For static IP, consult /etc/dhcpcd.conf and 'man dhcpcd.conf'

# Include files from /etc/network/interfaces.d:
source-directory /etc/network/interfaces.d

auto lo

auto eth0
allow-hotplug eth0
iface eth0 inet dhcp

allow-hotplug wlan1
iface wlan1 inet static
  address 192.168.100.1   # router setting
  netmask 255.255.255.0
  network 192.168.100.0
