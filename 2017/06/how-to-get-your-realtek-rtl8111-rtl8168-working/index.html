How to get your Realtek RTL8111/RTL8168 working

26 Jun 2017

원문 – How To get your Realtek RTL8111/RTL8168 working by Unixblogger
번역 – Yi Soo, (Thomas) An

Why do I need this driver anyway?

많은 분들이 제게 “이게 왜 필요하죠? 리눅스에서 지원하지 않나요?…” 라고 물어봅니다. 하지만 RTL8111/RTL8168은 오픈 소스가 아니에요. 이제 왜 리눅스 커널에 포함되지 않는지 아시겠죠?
그래서 우리가 직접 빌드해야 된답니다.

The installation methods

Ubuntu/Debian에서 디바이스 드라이버를 컴파일하는 방법에 대해 글을 쓴 이후로 많은 것이 변했습니다. 그래서 오늘은 드라이버를 설치하기 위한 방법 2개를 소개하고자 합니다.

The automatic way

NOTE: 힌트를 준 Liyu에게 고마움을 전합니다.
NOTE2: 인터넷 연결이 필요합니다. WLAN이나 USB 이더넷 카드를 이용해서 인터넷에 연결해주세요.

Ubuntu와 Debian의 저장소에는 우리가 설치할 드라이버가 포함되어 있습니다. Debian 유저라면 non-free 패키지 소스를 활성화(Enable) 해야하고 Ubuntu 유저라면 universe 패키지 소스를 활성화해야 합니다. /etc/apt/sources.list 를 root 권한과 함께 텍스트 에디터로 열어서 수정해야 합니다.

Debian 이용자

deb http://ftp.de.debian.org/debian/ jessie main contrib

위 라인을 다음과 같이 변경하고 저장합니다.

deb http://ftp.de.debian.org/debian/ jessie main contrib non-free

저장소를 업데이트합니다.

sudo apt-get update

Ubuntu 이용자

deb http://de.archive.ubuntu.com/ubuntu/ xenial main restricted

위 라인을 다음과 같이 변경하고 저장합니다.

deb http://de.archive.ubuntu.com/ubuntu/ xenial main restricted universe

저장소를 업데이트합니다.

sudo apt-get update

이제 준비가 되었으니 아래 명령어를 이용하여 디바이스 드라이버를 설치합니다.

sudo apt-get install r8168-dkms

CPU 종류에 따라 디바이스 드라이버를 빌드하는데 소요되는 시간이 있습니다.
자동화 방법의 장점은 Kernel 업데이트가 생기면 dkms 에 의해 r8168 드라이버도 업데이트됩니다. 빌드가 끝나고 이더넷 카드를 사용할 수 있어야 하는데 그렇지 않다면 재부팅합니다.

The manual way

NOTE: Manual 방식은 4.0 이상의 커널에서 테스트되었습니다.

1. 의존성 설치

우선 인터넷에 연결되어 있어야 합니다. 물론 필요한 패키지를 Debian/Ubuntu DVD에서 찾을 수도 있습니다.
의존성 패키지를 설치하기 위해 아래의 명령어를 입력합니다.

sudo apt-get install build-essential linux-headers-$(uname -r)

2. 드라이버 코드 다운로드

링크를 통하여 Realtek 홈페이지에서 공식 드라이버를 받을 수 있습니다. 커널 버전이 4.7 이상이면 LINUX driver for kernel up to 4.7를 선택하고 아니면 Linux driver for kernel 2.4.x (Support x86 and x64)를 선택합니다.

3. 블랙리스트에 r8169 드라이버 추가

시스템에 r8168 드라이버가 없으면 r8169를 사용하게 됩니다. 하지만 RTL8168 이더넷 카드에서 r8169 드라이버는 불안정합니다.1 r8169가 로드되는 것을 막기 위하여 블랙리스트를 이용합니다.

user@linux:~$ sudo sh -c 'echo blacklist r8169 >> /etc/modprobe.d/blacklist.conf'

4. 소스코드 압축해제

드라이버를 다운로드하고 나서 압축을 풉니다.

user@linux:~@ tar xfvj path/to/file.tar.bz2

NOTE: 소스코드 파일 이름은 다를 수 있으니 주의하세요.

5. 컴파일 및 설치

압축해제된 폴더로 이동합니다.

user@linux:~$ cd path/to/file

이제 스크립트를 이용하여 컴파일 및 설치 과정을 진행합니다.

user@linux:~/path/to/file$ sudo ./autorun.sh

드라이버 빌드가 끝나고 올바르게 설치되었다면 아래와 같이 출력되어야 합니다.

Check old driver and unload it.
rmmod r8169
Build the module and install

6. 드라이버 확인

마지막으로 드라이버가 올바르게 로드되었는지 확인합니다. 어떠한 오류도 없다면 아래와 같이 출력됩니다.

user@linux:~$ lsmod | grep r8168
r8168                 491520  0

이더넷 디바이스에서도 확인해봅니다.

user@linux:~$ sudo ethtool -i enp1s0
driver: r8168
version: 8.042.00-NAPI
firmware-version:
expansion-rom-version:
bus-info: 0000:07:00.0
supports-statistics: yes
supports-test: no
supports-eeprom-access: yes
supports-register-dump: yes
supports-priv-flags: no

로드되지 않았으면 시스템을 재부팅하고 다시 확인합니다.

That’s it
